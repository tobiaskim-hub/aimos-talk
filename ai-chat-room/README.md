# 💬 AI Multi-Agent Chat Room

> **확장 가능한 다중 AI 채팅방**  
> 👨‍💼 Tobi (사용자) + 👨 민수 (로컬 LLM) + 🤖 GenS (Claude) + 원하는 만큼 Bot 추가!

---

## 🎯 목적

**Ollama AI Agent 훈련 가이드의 실현 가능성을 검증하고, 여러 AI와 함께 토론하는 확장 가능한 채팅 UI**

### 기본 참가자:
1. **👨‍💼 Tobi** - 사용자 (당신!)
2. **👨 민수** - 로컬 LLM (Ollama Qwen 7B)
3. **🤖 GenS** - 클라우드 AI (Claude)

### ➕ 무제한 확장:
- 🤖 GPT-4, Gemini, Claude 3 등 추가 가능
- 👨 팀원, 전문가 페르소나 추가
- 🦾 특화된 AI 에이전트 추가

---

## ✨ 새로운 기능 (v2.0)

### 🆕 **다중 Bot 지원**
```
➕ "Bot 추가" 버튼으로 무제한 추가
✏️ 이름, 역할, 아이콘, 색상 커스터마이징
📍 메시지 위치 선택 (왼쪽/오른쪽)
🎨 8가지 색상 테마
```

### 🎨 **개선된 UI**
```
👥 동적 참가자 목록
🏷️ 역할 배지 표시
💬 개선된 메시지 버블
💾 대화 내보내기 (Markdown)
```

### 🚀 **확장성**
```
무제한 Bot 추가
각 Bot별 커스터마이징
실시간 참가자 업데이트
```

## 💬 사용 시나리오

### **시나리오 1: 4-Way 토론 (Tobi + 민수 + GenS + GPT-4)**

```
👨‍💼 Tobi (10:00):
Phase 1에서 10개 도구를 2주 만에 추가할 수 있을까?

---

👨 민수 (10:05):
가능합니다. 각 도구당 1~2일 소요.
edit_file부터 시작하는 걸 추천합니다.

---

🤖 GenS (10:08):
민수의 평가에 동의합니다.
다만 multi_edit는 3일 예상되므로 2주차로 미루세요.

---

🤖 GPT-4 (10:12):
두 분 의견 모두 타당합니다.
추가로 테스트 전략도 고려하세요:
- 각 도구 구현 후 즉시 통합 테스트
- Agent.ts의 프롬프트 조정 필요

---

👨‍💼 Tobi (10:15):
완벽합니다! 
민수 추천대로 edit_file부터 시작하고,
GenS 조언대로 multi_edit는 2주차로,
GPT-4 제안대로 즉시 테스트하겠습니다!
```

## 🎯 실전 활용 방법

### **1. Ollama 준비 (민수)**

```powershell
# Ollama 실행 확인
ollama list

# Qwen 모델 로드
ollama run qwen2.5:7b

# 테스트
ollama run qwen2.5:7b "안녕하세요, 민수입니다!"
```

### **2. 채팅방 열기**

```powershell
start C:\A-SDV-Platform\automotive-sdv-platform\ai-chat-room\index.html
```

### **3. 질문 시작**

```
1. "📋 질문1: 전체 로드맵" 버튼 클릭
2. 발신자: "👨‍💼 Tobi" 선택
3. "보내기" 클릭
```

### **4. 민수(Qwen) 답변 가져오기**

```powershell
# 별도 터미널에서
ollama run qwen2.5:7b

# 채팅방 질문 복사 → Ollama 붙여넣기
# 민수의 답변 복사

# 채팅방으로:
# 발신자: "👨 민수" 선택
# 답변 붙여넣기 → 전송
```

### **5. GenS 분석 추가**

```
발신자: 🤖 GenS

민수의 평가를 분석한 결과:

✅ 긍정적: [요약]
⚠️ 우려: [요약]
💡 제안: [제안]
```

### **6. 추가 Bot으로 확장 (선택)**

```
➕ Bot 추가 버튼 클릭
→ GPT-4, 철수, 영희 등 추가
→ 다각도 피드백 받기
```

## 💡 팁 & 트릭

### **✅ 효과적인 Bot 구성**

```
👨‍💼 Tobi (사용자)
  ↓ 질문
👨 민수 (빠른 첫 답변)
  ↓ 피드백
🤖 GenS (심층 분석)
  ↓ 추가 의견
🤖 GPT-4 (대안 제시)
  ↓ 최종 결정
👨‍💼 Tobi
```

### **✅ 색상 활용**

```
파란색: 공식/신뢰
주황색: 친근/로컬
초록색: 긍정/클라우드
빨간색: 주의/비판
보라색: 창의/UI
```

### **✅ 역할 분담**

```
민수: 기술적 타당성
GenS: 베스트 프랙티스
GPT-4: 대안 솔루션
전문가: 도메인 지식
```
